Practica 4: Microservicio de Autenticación JWT

Microservicio en Flask que utiliza **JWT (JSON Web Token)** para autenticar usuarios contra una base de datos. El microservicio permite registrar usuarios, iniciar sesión, generar tokens JWT y proteger rutas que solo se puedan acceder con un token válido.

Python Flask JWT REST API Docker Auth
Implementar un microservicio de autenticación seguro y escalable usando JSON Web Tokens (JWT) en Flask.
Esta práctica se centra en el desarrollo de un microservicio que gestiona la autenticación de usuarios mediante el estándar JWT, una pieza clave en la arquitectura de microservicios modernos. El uso de JWT permite la validación de identidad sin necesidad de mantener estados de sesión en el servidor.
Se desarrolló la lógica de registro, login y generación de tokens JWT en Python/Flask. Se implementó un decorador para proteger endpoints que requieren autenticación, se configuró la conexión a la base de datos para la gestión de usuarios, y se preparó la aplicación para contenerización con Docker.
Se logró un microservicio de autenticación desacoplado y seguro, capaz de emitir tokens que permiten a otros servicios validar la identidad del usuario sin necesidad de acceder a la base de datos de credenciales. La solución es eficiente y fácilmente integrable en arquitecturas distribuidas.
Autenticación basada en tokens (**JWT**), desarrollo de microservicios con **Flask**, manejo de **hashing de contraseñas**, protección de rutas (**decorators**), despliegue y orquestación con **Docker**.

Practica 5: Microservicio de Libros con Redis y JWT

Implementación de una **API RESTful para la gestión de un catálogo de libros**, donde la seguridad de todas las rutas CRUD está garantizada por **JSON Web Tokens (JWT)**. La innovación clave es la integración de **Redis** como una capa de soporte de alta velocidad para la gestión de tokens, incluyendo la **revocación de sesiones** (mediante *allowlist/denylist*) y la implementación de un mecanismo de **Refresh Token** para mejorar la experiencia de usuario y seguridad.

Python Flask JWT Redis Microservicios Seguridad
Diseñar e implementar un Microservicio de Catálogo de Libros, utilizando Redis como base de datos en memoria para gestionar y controlar de manera estricta el ciclo de vida y la revocación de tokens de seguridad JWT.
Para un sistema que requiere alta seguridad y un control estricto sobre las sesiones, la implementación del par Access/Refresh Token es esencial. Esta práctica extiende la funcionalidad JWT básica (Práctica 4) al integrar Redis, lo que permite la gestión de listas blancas/negras para tokens, posibilitando la revocación inmediata de sesiones, una característica vital que el JWT por sí solo no ofrece.
Se desarrolló el backend en Flask para el catálogo de libros, protegiendo todos los endpoints CRUD (`/books`). Se configuró la conexión a Redis para almacenar los tokens de acceso y refresh válidos (allowlist). Se implementó la lógica de revocación (marcando tokens en la denylist) y se creó un endpoint `/refresh` para emitir nuevos Access Tokens sin requerir el login nuevamente. La solución fue desplegada en Google Cloud Platform.
Se construyó una solución de autenticación robusta y desacoplada, apta para ser consumida por cualquier cliente o microservicio que necesite asegurar sus rutas. La adopción de JWT garantiza seguridad y mejora la escalabilidad al eliminar la necesidad de almacenamiento de sesiones en el servidor.
Integración de **Redis** para gestión de caché y sesiones, implementación de **Access/Refresh Tokens** para seguridad, gestión de **revocación de tokens (allowlist/denylist)**, desarrollo de endpoints **RESTful seguros**, y orquestación de un ecosistema de microservicios (Flask + Redis).

Python TKinter MVC Architecture REST Client JWT Handling
Desarrollar una aplicación gráfica de escritorio (GUI) en Python (Tkinter) capaz de consumir de forma segura los endpoints de microservicios REST, implementando un manejo robusto de la autenticación y el monitoreo de servicio.
Aunque las GUIs modernas suelen ser web, este proyecto demuestra la capacidad de Python para crear clientes de escritorio ligeros y potentes que interactúan con arquitecturas de microservicios. La arquitectura MVC asegura la separación de la lógica de negocio (manejo de JWT y APIs) de la interfaz de usuario (Tkinter).
El cliente fue construido bajo el patrón MVC, separando la interfaz visual de la lógica de red y del modelo de datos. Se logró un cliente de escritorio funcional y bien estructurado que sirve como una prueba de concepto sólida para la integración de Tkinter con servicios web modernos. El uso de MVC, el manejo de JWT y el monitoreo activo demuestran un alto nivel de competencia en el desarrollo de clientes para microservicios.
Implementación de arquitectura **MVC** en Python, desarrollo de **GUIs de escritorio con TKinter**, manejo de **peticiones HTTP seguras**, gestión del ciclo de vida de **JWT** en una aplicación cliente, y aplicación de **patrones de diseño de software** para sistemas distribuidos.

Tarea 5: Comparativa de Rendimiento (Redis vs MariaDB)

Aplicación de consola en **Python** diseñada para **comparar el rendimiento** de las bases de datos **Redis** (In-Memory) y **MariaDB** (Relacional) en escenarios de alto volumen de datos. La simulación mide los tiempos de respuesta para operaciones de **inserción y lectura** (registro y autenticación de usuarios), ofreciendo una perspectiva práctica sobre cuándo elegir cada tecnología para optimización de acceso a datos.

Python Redis MariaDB Benchmarking Bases de Datos
Desarrollar una herramienta de benchmarking en Python para medir y comparar el rendimiento de Redis y MariaDB en operaciones críticas de lectura y escritura (simulación de autenticación y registro).
En la arquitectura de microservicios, la elección de la base de datos es crucial para la eficiencia. Esta actividad se centró en cuantificar las diferencias de latencia entre una base de datos en memoria (Redis), ideal para caching y sesiones, y una relacional persistente (MariaDB), midiendo el impacto real en el rendimiento.
Se implementó un script en Python que realiza series de pruebas controladas. El programa gestiona las conexiones a ambos sistemas (MariaDB para persistencia simulada y Redis para caché/sesiones simuladas), ejecuta operaciones repetitivas (INSERT/SET y SELECT/GET) y utiliza la librería `time` para registrar métricas detalladas de tiempo de ejecución para cada sistema.
La actividad demostró que, mientras MariaDB es esencial para la persistencia transaccional, Redis ofrece órdenes de magnitud de mayor velocidad en operaciones de lectura y escritura simples, confirmando su rol como la capa de acceso rápido para tokens, sesiones o datos de baja latencia en arquitecturas de integración modernas.
Conexión y manipulación de datos con **MariaDB y Redis en Python**, técnicas de **benchmarking y medición de rendimiento**, comprensión práctica de las ventajas de las bases de datos **In-Memory** frente a las **Relacionales**, y optimización de flujos de trabajo de **autenticación/registro**.
