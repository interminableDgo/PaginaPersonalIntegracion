Práctica Guiada

Proyecto desarrollado durante el curso de integración.

Practica 1: Clasificador de Servicios de la Nube

Proyecto que inicia con un clasificador simple de servicios usando la API de OpenRouter y evoluciona a una versión avanzada con mejoras en el procesamiento de texto y optimización de llamadas a la API.

Python OpenRouter API Requests API Optimization
Clasificar servicios de la nube en categorías utilizando una API de IA.
Este ejercicio introduce la integración con modelos de lenguaje (OpenRouter) para automatizar la categorización de descripciones de servicios cloud.
Se implementó un clasificador en Python que consume la API, define el prompt, maneja variables de entorno y estructura las respuestas para optimizar costo y latencia.
Se obtuvo un clasificador funcional y escalable que sirve como base para futuras mejoras y nuevas categorías.
Integración de APIs de IA, manejo de variables de entorno, clasificación automatizada de texto, arquitectura escalable y optimización de consumo de API.

Practica 2: Chat con Ollama y DeepSeek

Es una aplicación web en Flask que se conecta a Ollama para usar modelos de lenguaje locales (como DeepSeek, Llama 3 y Phi 3), con interfaz tipo chat, API RESTful, despliegue en Docker, manejo de errores e indicadores de estado, demostrando un desarrollo web moderno e integración fluida con IA local.

Python Flask Ollama DeepSeek Docker HTML/CSS/JS
Desarrollar una app de chat web que use modelos locales vía Ollama con backend Flask.
Se plantea una solución on-prem/local para interacción con LLMs (DeepSeek, Llama 3, Phi 3), evitando dependencia de servicios externos.
Se construyó una API REST con Flask, interfaz tipo chat en HTML/CSS/JS con selección de modelo y estados, y se orquestó con Docker para facilitar ejecución y despliegue.
La app entrega un chat funcional, extensible y contenerizado, listo para incorporar mejoras como streaming, historiales y autenticación.
Integración con modelos de IA locales, desarrollo de aplicaciones web con Flask, contenerización con Docker, APIs RESTful, interfaz de usuario moderna y responsiva, manejo de modelos de lenguaje local.

Practica 3: Biblioteca XML con XSLT

Sistema de gestión de biblioteca implementado con XML para almacenar datos de libros y XSLT para transformación y presentación. Incluye funcionalidad de búsqueda por ISBN y generación de vistas estructuradas de la información bibliográfica.

XML XSLT CSS Biblioteca Digital
Crear un sistema de biblioteca digital usando XML para almacenar datos de libros y XSLT para transformación y búsqueda por ISBN.
Este ejercicio introduce el manejo de datos estructurados con XML y su transformación mediante XSLT para crear interfaces de consulta y presentación de información bibliográfica.
Se implementó un archivo XML con estructura de biblioteca, hojas de estilo XSLT para transformación de datos, CSS para presentación visual, y funcionalidad de búsqueda por ISBN con filtrado dinámico.
Se logró un sistema funcional de biblioteca digital que demuestra el poder de XML/XSLT para manejo de datos estructurados y generación de interfaces web dinámicas.
Estructuración de datos con XML, transformación con XSLT, diseño de interfaces con CSS, búsqueda y filtrado de información, gestión de bibliotecas digitales.

Tareas de Investigación

Análisis y documentación de conceptos fundamentales en tecnologías cloud y servicios web.

Tarea 1: Modelos de Servicios en la Nube

Análisis exhaustivo de los diferentes modelos de servicios en la nube: SaaS (Software as a Service), IaaS (Infrastructure as a Service), PaaS (Platform as a Service) y FaaS (Function as a Service). Incluye diagramas comparativos, casos de uso, ventajas y desventajas de cada modelo, así como ejemplos prácticos de implementación en el mundo real.

Cloud Computing SaaS IaaS PaaS FaaS Arquitectura Cloud
Comprender y documentar los diferentes modelos de servicios en la nube, sus características distintivas, casos de uso y diferencias arquitectónicas.
Esta investigación profundiza en los fundamentos de la computación en la nube, explorando los cuatro modelos principales de servicios y su evolución en el ecosistema tecnológico actual.
Se realizó un análisis comparativo detallado de cada modelo, incluyendo diagramas de arquitectura, tablas comparativas de características, ejemplos de proveedores líderes y casos de uso específicos para cada tipo de servicio.
La investigación revela la importancia de seleccionar el modelo de servicio adecuado según las necesidades específicas del proyecto, considerando factores como escalabilidad, control, costos y complejidad de gestión.
Modelos de servicios cloud, arquitecturas distribuidas, escalabilidad horizontal y vertical, gestión de infraestructura, plataformas de desarrollo, funciones serverless, y estrategias de migración a la nube.

Tarea 2: Miniensayo: ¿Cloud-native o cloud-enabled?

Análisis de una aplicación real para determinar si es cloud-native o cloud-enabled, evaluando sus características arquitectónicas, patrones de diseño, y nivel de integración con servicios cloud. El miniensayo incluye comparaciones, casos de uso y conclusiones sobre las ventajas y desventajas de cada enfoque.

Cloud-native Cloud-enabled Análisis de Arquitectura Minienseyo Aplicaciones Cloud
Analizar una aplicación real para determinar si es cloud-native o cloud-enabled, evaluando sus características arquitectónicas y nivel de integración con servicios cloud.
Este miniensayo explora las diferencias entre aplicaciones cloud-native y cloud-enabled, analizando una aplicación real para identificar sus patrones arquitectónicos y nivel de adopción de tecnologías cloud.
Se realizó un análisis detallado de una aplicación seleccionada, evaluando sus características como escalabilidad, resilencia, uso de servicios cloud, contenerización, y patrones de diseño para clasificarla como cloud-native o cloud-enabled.
El análisis revela las ventajas y desventajas de cada enfoque, proporcionando insights valiosos sobre cuándo y cómo adoptar estrategias cloud-native vs cloud-enabled según las necesidades del proyecto.
Cloud-native vs cloud-enabled, análisis de arquitecturas, patrones de diseño cloud, escalabilidad, resilencia, contenerización, servicios cloud, y estrategias de migración.

Tarea 3: Arquitectura Cloud para Banca Móvil

Simulación del diseño de una arquitectura cloud para una aplicación de banca móvil utilizando una nube híbrida y funciones serverless. Incluye diagrama arquitectónico, explicación de la solución y código backend básico implementado en el repositorio de GitHub.

Cloud Architecture Hybrid Cloud Serverless Banca Móvil Backend GitHub
Diseñar y simular una arquitectura cloud híbrida y serverless para una aplicación de banca móvil, mostrando el diagrama y el backend básico.
Esta tarea explora cómo una solución de banca móvil puede aprovechar una nube híbrida y funciones serverless para lograr escalabilidad, seguridad y eficiencia operativa.
Se diseñó un diagrama arquitectónico que integra servicios en la nube pública y privada, junto con funciones serverless para el procesamiento de transacciones. El backend básico está disponible en el repositorio de GitHub, mostrando la estructura y lógica principal de la solución.
La arquitectura propuesta permite una alta disponibilidad, escalabilidad y seguridad, aprovechando lo mejor de ambos mundos (cloud pública y privada) y la eficiencia de las funciones serverless.
Diseño de arquitecturas híbridas, uso de serverless, integración de servicios cloud, seguridad en banca móvil, despliegue de backend básico.
